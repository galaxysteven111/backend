# 手機版 UI 評估 Prompt

請作為一位資深的移動端 UI/UX 設計專家，對以下捐飯盒平台的手機版界面進行全面評估，並提供具體的改進建議。

## 平台背景
這是一個香港的飯盒捐贈平台，連接願意分享食物的捐贈者與需要幫助的人。目標用戶包括：
- 想要捐贈多餘飯盒的愛心人士
- 需要飯盒幫助的弱勢群體
- 關注食物浪費和社區關愛的用戶

## 需要評估的方面

### 1. 視覺層次和布局
- 信息架構是否清晰？
- 內容層次是否合理？
- 視覺焦點是否突出？
- 空白空間使用是否得當？

### 2. 導航和可用性
- 底部導覽是否易於使用？
- 導航邏輯是否清晰？
- 用戶能否快速找到所需功能？
- 操作流程是否順暢？

### 3. 觸控體驗
- 按鈕大小是否適合手指點擊（建議至少 44x44px）？
- 觸控目標間距是否足夠？
- 反饋是否及時明確？
- 是否有誤觸風險？

### 4. 內容可讀性
- 字體大小是否適合手機閱讀（建議正文至少 16px）？
- 行距和字距是否舒適？
- 對比度是否足夠？
- 文字是否過於密集？

### 5. 響應式設計
- 在不同屏幕尺寸下是否正常顯示？
- 橫屏和豎屏是否都適配？
- 是否考慮了不同設備的差異？

### 6. 用戶流程
- 「我要捐飯盒」和「我要取飯盒」兩個流程是否清晰？
- 用戶能否快速理解如何使用平台？
- 關鍵操作是否容易找到？

### 7. 視覺設計
- 顏色搭配是否適合慈善平台？
- 是否給人溫暖、可信的感覺？
- 圖標和插圖是否清晰易懂？
- 整體風格是否統一？

### 8. 性能優化
- 加載速度是否合理？
- 動畫是否流暢？
- 是否有不必要的視覺元素？

## 請提供
1. **優點**：列出 3-5 個做得好的地方
2. **問題**：列出 5-10 個需要改進的問題
3. **建議**：針對每個問題提供具體的改進建議
4. **優先級**：標註哪些改進最緊急/重要
5. **最佳實踐**：提供移動端 UI 設計的最佳實踐建議

## 評估標準
- 易用性（Usability）
- 可訪問性（Accessibility）
- 視覺吸引力（Visual Appeal）
- 用戶體驗（User Experience）
- 專業度（Professionalism）

請詳細分析並提供可執行的改進方案。
