# 代碼審查檢查清單

## 🔍 給 Claude AI 的審查指南

請按照以下清單系統性地審查這個「香港捐飯盒平台」項目。

---

## 📋 1. 代碼結構審查

### 前端結構
- [ ] `frontend/src/` 目錄結構是否合理
- [ ] 組件是否按功能模塊組織
- [ ] 是否有重複代碼需要提取
- [ ] 工具函數是否放在合適的位置

### 後端結構
- [ ] `backend/src/` 目錄結構是否合理
- [ ] 路由組織是否清晰
- [ ] 中間件是否正確使用
- [ ] 數據庫配置是否合理

---

## 🐛 2. Bug 檢查

### 前端 Bug
- [ ] 底部導覽列是否在所有頁面正確顯示
- [ ] 表單提交是否有驗證
- [ ] API 錯誤是否正確處理
- [ ] 路由跳轉是否正常
- [ ] 狀態管理是否正確
- [ ] 響應式布局是否有問題

### 後端 Bug
- [ ] API 端點是否正確
- [ ] 數據庫查詢是否有錯誤
- [ ] 認證中間件是否正確
- [ ] 錯誤處理是否完善
- [ ] 數據驗證是否充分

---

## 🔒 3. 安全性審查

### 認證與授權
- [ ] JWT 實現是否安全
- [ ] 密碼是否正確加密（bcrypt）
- [ ] 認證中間件是否正確使用
- [ ] 是否有權限檢查

### 輸入驗證
- [ ] 所有用戶輸入是否驗證
- [ ] SQL 注入防護（Knex 參數化查詢）
- [ ] XSS 防護（清理用戶輸入）
- [ ] CSRF 防護

### API 安全
- [ ] CORS 配置是否正確
- [ ] 是否有速率限制
- [ ] 敏感信息是否洩露

## 🎯 優先級建議

### P0（立即修復）
1. 安全性漏洞
2. 導致功能無法使用的 bug
3. 數據丟失風險

### P1（重要改進）
1. 用戶體驗問題
2. 性能問題
3. 代碼質量問題

### P2（優化）
1. 代碼重構
2. 功能增強
3. 文檔完善

---

## 💡 改進建議格式

對於每個發現的問題，請提供：

```markdown
### [問題標題]

**嚴重程度**：P0/P1/P2  
**位置**：文件路徑  
**問題描述**：詳細說明  
**影響**：對用戶/系統的影響  
**解決方案**：如何修復  
**代碼示例**：修復後的代碼
```
